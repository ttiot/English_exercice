Chiffrement des sauvegardes

La clé est stockée chiffrée en base de données (Fernet) et exportée vers /data/.backup_key à la sauvegarde. Le conteneur de backup lit ce fichier pour chiffrer les exports SQLite en AES-256-CBC.

{# ── Aide ── #}

Comment ça marche

  1. Cliquez Générer une clé aléatoire — la clé apparaît dans le message flash.
  2. Copiez-la dans votre gestionnaire de mots de passe (elle ne sera plus affichée).
  3. Le fichier /data/.backup_key est automatiquement mis à jour.
  4. Le conteneur backup lit ce fichier à chaque cycle et chiffre l'export SQLite.
  5. Pour restaurer, déchiffrez avec openssl enc -d -aes-256-cbc -pbkdf2 -pass pass:<clé> -in backup.sqlite.enc -out restored.sqlite.
